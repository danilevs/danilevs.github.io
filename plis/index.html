Лабораторная работа № 2  

Моделирование систем на кристалле с процессорным ядром MicroBlaze.

Цель работы: изучение принципов моделирования систем на кристалле с процессорным ядром MicroBlaze с применением системы моделирования ISim.

Основные теоретические сведения.

Моделирование систем с процессорными ядрами в среде XPS.

Запускается XPS (Xilinx Platform Studio), в открывшемся окне выбирается  Create New Project Using Base System Builder, а затем в окне Create New XPS Project Using BSB Wizard указывается путь к проекту и имя (например, system.xmp). Проект следует создавать в отдельной папке, эту папку надо создать (см. рис. 1). Кроме того по указанию преподавателя надо выбрать тип системы: AXI System или PLB System (по умолчанию предлагается AXI System).

Для автоматизации запуска можно выполнить Project ⇒ Build Automatically. В этом случае трансляция выполняется автоматически при сохранении файла программы (по умолчанию это установлено). Изменить текст программы можно двойным щелчком мыши на строке helloworld.c в левой части. Если программа содержит включаемые файлы, то они отображаются в правой части, двойным щелчком мыши можно открыть выбранный файл.

Для начального варианта можно использовать следующую программу.

#include "xgpio_l.h"

#include "xparameters.h"

int main()

{int i,v=0;

  XGpio_WriteReg(XPAR_LEDS_4BIT_BASEADDR, XGPIO_TRI_OFFSET, 0);

  while(1) {

    //i=Xil_In32(XPAR_PUSH_BUTTONS_POSITION_BASEADDR);

    v=v+3;

    Xil_Out32(XPAR_LEDS_4BIT_BASEADDR, v);

  }

  return 0;

}

Данная программа выдает в выходной порт значения 3, 6, 9, …. После ввода и сохранения программы выполняется трансляция и генерирование исполняемого файла p0_nexy2_0.elf.

Не закрывая SDK можно переключиться в XPS. Здесь необходимо указать путь к исполняемому файлу p03_nexy2_0.elf. Для этого в поле Project надо найти строку Sim Executable: и нажать на ней правой клавишей мыши (см. рис. 6). Здесь следует выбрать Browse и указать путь к файлу, например …\SDK\SDK_Export\workspace\p0_nexy2_0\Debug\p0_nexy2_0.elf

На диаграмме виден сигнал общего сброса (fpga_0_rst_1_…) в начальный момент времени (активное значение - единица). Программа обработки начинает работать примерно с отметки времени 5 мкс, так как с момента примерно 5.4 мкс начинается изменение выходного сигнала fpga_0_leds_…. До момента 5.4 мкс работает программа операционной системы, которая проводит все необходимые настройки. Изменение значений выходного сигнала совпадает с предполагаемыми значениями. Синхросигнал fpga_0_clk_1_sys… изменяется слишком часто для данного масштаба и поэтому его колебания не видны.

Для получения листинга можно также в окне XMD Console выполниеть команду подключения connect mb sim, а затем команду загрузки программы dow <filename.elf>, причем путь указывается полный (слэши надо заменить). Затем выполняется команда дизассемблирования, ПРИМЕР: dis 0x200 0x300 (0x300 команд с 200-го адреса).

Отладчик запускается командой gdb: mb-gdb  executable-file (путь указывается полный, разделители между именами папок - /), и наконец, disassemble /m main.

Для получения кодов команд: disassemble /r main.

Моделирование системы с подчиненным процессорным блоком

Создание системы с подчиненным процессорным блоком в системе ISE 14.7 рассмотрим на примере ПЛИС Spartan-6.

Структура системы с подчиненным процессорным блоком показана на рис. 15. Внешние сигналы системы поступают в главный модуль, они могут передаваться в процессор непосредственно или после обработки – это зависит от выполняемой задачи. Внешние сигналы процессорного блока являются внутренними сигналами главного модуля.

Рис. 15.

Сначала в системе ISE создается проект. После создания проекта генерируется процессорный блок на основе ядра MicroBlaze. Для этого в системе ISE из основного меню выполняется следующая последовательность: Project ⇒ New Source. В открывшемся окне выбирается пункт Embedded Processor, в поле File Name задается имя, например, system4, после чего нажимается клавиша Next, а в следующем окне - Finish. В результате запускается XPS (Xilinx Platform Studio), а далее для создания процессорного блока надо следовать указаниям мастера Base System Builder, при этом по указанию преподавателя надо выбрать тип системы: AXI System или PLB System. В дальнейшем изменяется полярность сброса и устанавливается объем памяти - не менее 16 KB.

В следующем окне задаются внешние устройства После этой операции фрагмент окна внешних устройств примет вид, показанный на рис. 16.

Рис. 16. Окно внешних устройств.

Следует отметить, что сигналы ввода-вывода процессорного блока используются не для подключения кнопок и светодиодов, а в качестве параллельных портов общего назначения.

Далее запускается SDK. При запуске SDK необходимо указать расположение проекта, например: …\system4\SDK\SDK_Export\workspace_4. При этом надо найти свой текущий проект, так как Вам, вероятнее всего будет предложен какой-то другой проект. После этого открывается окно SDK, в котором надо создать новый программный проект. Исходный модуль программы задается следующим образом:

 

#include "xgpio_l.h"

#include <stdio.h>

#include "xparameters.h"

void print(char *str);

int main()

{int v=0x0;  

 XGpio_WriteReg(0x40000000, XGPIO_TRI_OFFSET, 0);

  while(1) {

    v=v+3;

    Xil_Out32(0x40000000, v);

  }

  return 0;

}

При необходимости адреса и их правильное значение можно уточнить в файле xparameters.h. Программа транслируется, если есть ошибки, их надо исправить.

После создания аппаратной платформы процессорного ядра и программы можно закрыть SDK и XPS и вернуться в ISE. После этого в окне Hierarchy надо выделить строку с процессорным блоком, а в окне Processes – активизировать строку Generate Top HDL Source, после чего проект примет вид, приведенный на рис. 17.

Рис. 17. Система с процессорным блоком.

После этого программу необходимо подключить к процессорному блоку, эта процедура выполняется в системе ISE. Программа (она имеет расширение elf) может располагаться в папке

…system4\SDK\SDK_Export\workspace_4\hello_world_0\Debug\.

После подключения окно Hierarchy примет вид, показанный на рис 18..

Рис 18. Окно Hierarchy

Далее создается файл внешних воздействий и выполняется моделирование. При создании файла внешних воздействий следует проверить наличие сдвига сигналов CLK_Р и CLK_N.

После внесения изменений в программе надо выполнить Project => Cleanup Project Files.  

Порядок выполнения работы

1. Изучить принципы моделирования систем на кристалле с процессорным ядром Microblaze на примере рассмотренной выше программы. Оптимизацию программы при трансляции отключить.

Изменить программу так, чтобы вместо оператора v=v+3 использовался оператор v=v+5. Внести изменения в программу и выполнить моделирование.

Включить оптимизацию и повторить моделирование.

2. Изменить начальный вариант программы (по п. 2), в программу добавить оператор ввода. Введенное значение увеличить на m (номер варианта) и вывести в устройство вывода. Оптимизацию программ при трансляции отключить.

Изменяя время переключения входных сигналов, определить минимальное и максимальное время (в тактах синхросигнала)  реакции программы на это изменение.

3. Изменить программу в процессоре по заданию преподавателя (например, добавить блок пользователя), получить временные диаграммы.  

4. Повторить п.п. 1-3 для системы с подчиненным процессорным ядром.
